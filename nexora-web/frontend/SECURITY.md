# Security Policy

## Supported Versions

Only the latest stable release of **Nexora** is actively supported with security updates.

| Version | Supported |
|--------|-----------|
| Latest | ✅ |
| Older | ❌ |

---

## Reporting a Vulnerability

Security is a top priority for Nexora.

If you discover a security vulnerability, please **do not open a public issue**.

Instead, report it responsibly by contacting the project maintainer directly.

### What to Include
Please include as much detail as possible:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested mitigation (if known)

---

## Responsible Disclosure

We ask that you:
- Give us reasonable time to investigate and fix the issue
- Avoid public disclosure until a fix is released
- Act in good faith and avoid exploitation

---

## Response Process

Once a vulnerability is reported:
1. We will acknowledge receipt of the report
2. The issue will be investigated and verified
3. A fix will be developed and tested
4. A security update will be released as soon as possible

---

## Security Best Practices

Nexora follows security best practices, including:
- Secure authentication and authorization
- Input validation and sanitization
- Regular dependency updates
- Principle of least privilege

---

## Security Updates

Security patches will be released as part of regular updates or as hotfixes when required.

---

## Thank You

We appreciate the efforts of security researchers and contributors who help keep **Nexora** safe and reliable.

Your responsible disclosure helps protect the entire community.
